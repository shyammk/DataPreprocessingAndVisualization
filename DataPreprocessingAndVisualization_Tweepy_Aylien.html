[1.] Selection of Relevant Web-APIs

We have used four web-APIs in this project.

(a) TMDb API - https://developers.themoviedb.org/3/getting-started/introduction

  • This is the official API of the The Movie Database (TMDb). For the freely available version, an API key is required, which is provided upon signup and a small amount of rate limiting is enforced. The current limits are 40 requests every 10 seconds and are limited by IP address, not API key. The only format supported by this API is JSON.

(b) OMDb API - http://www.omdbapi.com/

  • Although unofficial, the OMDb (The Open Movie Database) API is a RESTful web service to obtain movie information from IMDb. An API key is provided after signup, to fetch the movie data and it supports both XML and JSON formats. In our case, we have used the JSON format. For the free version, the rate limit is 1000 requests per day.

(c) Tweepy Standard Search - https://developer.twitter.com/en/docs/tweets/search/api-reference/get-search-tweets.html

  • The wrapper for official Twitter API (RESTful web service), to search for a specific term and extract a certain number of tweets about it from twitter. Four API keys are required to use this API and extract tweets, and is provided only if the user has a valid twitter account. The API supports multiple formats, but we have chosen to use JSON format in our project. Rate limiting of the API is primarily on a per-user basis or more accurately described, per user access token. When using application-only authentication, rate limits are determined globally for the entire application.

(d) AYLIEN Sentiment Analysis API - http://docs.aylien.com/textapi/#using-the-api

  • AYLIEN API is effective in bringing the power of Natural Language Processing to textual content, in our case, tweets. Using this API, we can detect the sentiment of a document, in terms of its polarity ("positive" or "negative" or "neutral") and subjectivity ("subjective" or "objective"). An application key and ID are required to extract the sentiments using this API and would be provided only upon signing up. The API returns both XML and JSON, and we use JSON. Rate limiting of this API is 1000 calls per day and 120 calls per minute.

def getIMDbMovieDetails(tmdb_id, imdbParamList):
    imdb_details_list = []
    # Retrieve the id of the movie in IMDb
    imdb_id =  getIMDbMovieID(tmdb_id)
    if(imdb_id is None):
        # If no IMDb ID is obtained (or it is of None type), then set 'N/A' as the default value for all the parameters.
        for param in imdbParamList:
            imdb_details_list.append("N/A")
    else:
        # Form the URL, fetch the details of the movie from IMDb and append them to the list.
        omdb_get_movie_details_url = omdb_get_movie_details_url_head+imdb_id+omdb_get_movie_details_url_tail
        imdb_details_json = getApiURLJsonData(omdb_get_movie_details_url)
        if(imdb_details_json['Response']=="True"):
            for param in imdbParamList:
                imdb_details_list.append(imdb_details_json[param])
        else:
            for param in imdbParamList:
                imdb_details_list.append("N/A")
        
    return imdb_details_list

# Function to analyze the tweet and determine the sentiment using AYLIEN API.
def findTweetSentiment(tweetContent):
    sentimentPolarity=""
    
    # Remove extra spaces or newlines around the text and use 'utf-8' encoding
    tweetContent = tweetContent.encode('utf-8').strip()
    
    # Reject tweets which are empty
    if len(tweetContent) != 0:
        # Invoke the AYLIEN Text API
        sentiment = ay_client.Sentiment({'mode':'tweet', 'text': tweetContent})
        # Get the polarity/nature of the sentiment conveyed by the tweet
        sentimentPolarity = sentiment['polarity']
    return sentimentPolarity

def create_file (file_name, headerList):
    # Create (open) the file. The parameter newline = '', in order to prevent blank lines being written.

Normalize/Scale the Numeric Values¶

All the numeric values in the dataframe must be normalized or brought to the same range, for effective results to be obtained as a result of the analysis.

__We can see that, all the values in the columns 'Runtime(mins)', 'Revenue($)', 'Popularity', 'VoteCount' & 'VoteAverage' are not in the same range. Hence, we apply min-max normalization on the values in all these columns, except 'VoteAverage'. To do this, we take the minimum and maximum values in each of these columns and, then apply the formula (val-min)/(max-min) to normalize the values.
In case of the column 'VoteAverage', we just divide each of its values by 10, since it is already in the range of 0-10.__</font>

Conclusions Drawn

In the above steps, we used multiple web-APIs to retrieve the data related to movies, parsed & combined them to a single CSV file, pre-processed the data and finally, created some visualizations based on our analysis.

Through a series of steps, we were able to analyze and understand the following:

  • In which month of a year, the most number of movies are released.
  • Identify the pattern of distribution of the runtime of a movie across our dataset.
  • Nature of relationship between the Popularity & Revenue of a movie.
  • How the Revenue and Average of Votes, vary across different months in a year.
  • Identify the pattern of Revenue Distribution across different months, for every Year of Release.
  • The top-20 Directors and Ratings for their movies, based on the Popularity of the movies.
  • How the different attributes of our dataset are connected to each other, based on the Ratings(certifications) of the movies.
  • The number of movies per Rating/Certification.

The point to note here is that, these conclusions are tentative and may be vary (or) remain same/smiliar when analyzed on huge datasets. At the moment, these results are based on 300 popular movies collected using multiple APIs.
